1.웹 애플리케이션은 MVC 디자인 패턴을 사용하여 개발한다. MVC는 모델(Model), 뷰(View), 컨트롤러(Controller)를 의미한다.

2.뷰(View)는 클라이언트가 보는 화면으로서 서비스를 요청하고 서비스 응답 결과를 처리한다.

3.컨트롤러(Controller)는 뷰에서 들어온 요청을 받아 서비스 처리 메소드를 호출후 응답 뷰로 이동한다.

4.모델(Model)의 Service는 클라이언트가 요청한 서비스를 처리하는 객체이다.

5.모델(Model)의 DAO는 데이터베이스에 관련된 기능을 처리하는 객체이다.

6.뷰는 JSP, HTML, CSS, JavaScript 등으로 구현된다.

7.컨트롤러는 JSP 또는 Servlet으로 구현된다.

8.MVC Model1은 JSP로 컨트롤러를 구현한 디자인 패턴이다.

9.MVC Model2은 Servlet로 컨트롤러를 구현한 디자인 패턴이다.

10.Service, DAO 객체인 모델은 POJO(Plain Old Java Object)로 구현된다.

11.뷰와 컨트롤러를 프레젠테이션 계층, Service를 비즈니스 계층, DAO를 영속 계층으로 구분한 것을 3계층 아키텍처라고 한다.

12.프레임워크는 각 계층별로 지원되며 개발을 쉽고 빠르게 할 수 있도록 한다.

13.프런트 컨트롤러 디자인 패턴은 뷰에서 들어온 모든 요청을 대표 컨트롤러가 받아 처리한 후 실제 컨트롤러를 실행하는 구조이다.
